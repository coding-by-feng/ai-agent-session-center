# Restart / Reload Audit — Fix Plan & Checklist

Generated from a verified multi-agent audit (66 confirmed, 13 refuted). Branch: `fix/restart-reload-audit`.
Status legend: [ ] pending · [x] done · [~] partial/deferred

## Root cause
Three uncoordinated restore actors on restart: (1) server boot respawn `respawnSshTerminals` (server/index.ts), (2) client workspace import clear-all+recreate (workspaceSnapshot.ts via useWorkspaceAutoLoad), (3) pinned respawn timer (pinnedRespawn.ts). They double-resume every session and make the RestorePicker unenforceable.

## ✅ Fixed in this session (branch `fix/restart-reload-audit`) — tsc clean, 44 unit tests green, validated live via Playwright

**KEYSTONE — server boot respawn gated off by default** (`server/index.ts`): `respawnSshTerminals` now runs only when `AASC_SERVER_AUTORESPAWN=1` (headless opt-in). The client RestorePicker is authoritative. This single change resolves/dissolves: H1 (dual uncoordinated paths), the "Resume nothing unenforceable" pair, "server resurrects ALL regardless of picker", floating-fork `--continue` hijack, "duplicate claude when process survived", divergent resume builder, respawn-loop pendingLink theft, respawn retry PTY leaks, and the clear-all↔respawn orphan-PTY startup races.

A paused loop no longer comes back armed. (Resolves the H "automation reset" pair.)
